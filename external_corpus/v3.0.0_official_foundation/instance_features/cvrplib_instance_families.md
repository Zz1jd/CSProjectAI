CVRPLIB is not one monolithic benchmark. The official index groups instances into named families such as A, B, E, F, M, P, CMT, tai, Golden, Li, X, AGS, DIMACS, XML, and XL, each reflecting different sizes or historical benchmark lines. A governed instance-feature document should preserve those family labels because they are the stable handles used when papers or solvers report gaps. The site also couples instance downloads with best-known-solution tables, which makes family identity part of the evaluation contract rather than metadata decoration.
